# FallGuard 固定导航界面改版设计

## 目标

在不改变 YOLO Pose、ST-GCN 推理、历史记录格式和视频回放功能的前提下，重新设计
Tkinter 监控后台。导航在页面切换和窗口缩放时保持固定尺寸，实时检测与历史记录页面使用
一致、克制的视觉层级。

## 布局

- 顶部保留 64px 系统栏，左侧显示 FallGuard 与“跌倒监测后台”，右侧显示双模型状态。
- 系统栏下使用自绘顶部导航，不再使用 `ttk.Notebook` 原生标签。
- “实时检测”和“历史记录”按钮均固定为 `112×44`，字体、内边距和位置不随选中状态变化。
- 选中状态仅通过文字颜色与底部 2px 蓝色指示线表达。
- 页面内容位于独立容器中，通过 `tkraise()` 切换，不销毁或重建导航。

## 实时检测页

- 视频为主视觉区域，右侧保留当前状态、跌倒概率、处理进度和检测事件四项数据。
- 移除粗白色外边框，使用深蓝灰背景和 1px 分隔线组织层级。
- 时间轴紧邻视频区域，蓝色表示处理进度，红色只表示跌倒区间。
- 视频路径与三个操作按钮保持在底部，避免参数信息进入主工作区。

## 历史记录页

- 历史表格使用固定表头、轻分隔线和单行选中背景。
- 下方回放区域与实时视频区域使用相同宽高关系；右侧只显示记录信息和播放按钮。
- 历史时间轴继续读取 `events.json`，显示红色跌倒区间与当前播放位置。

## 视觉规范

- 使用一套深蓝灰表面层级、一个蓝色操作色和一个红色告警色。
- 删除不承担交互含义的卡片边框、厚描边和重复标题。
- 仅使用 Microsoft YaHei UI，标题、数据和辅助文字采用三档字号。
- 不新增动画；仅保留导航指示线、实时帧刷新和播放位置变化三种状态反馈。

## 验收

- 两个导航按钮在切换前后具有相同的请求宽高与实际宽高。
- 窗口缩放时导航栏高度固定，内容区域承担伸缩。
- 实时检测、跌倒区间、历史列表和历史回放功能保持可用。
- 全量 pytest、Ruff 和 mypy 通过，并重新生成实时页和历史页截图。
